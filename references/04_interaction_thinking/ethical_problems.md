# 互動思考 / 解決道德問題（code 語境）

> 主分類色：綠色系 `#0A8A4A`。技術決策的倫理面——處理隱私、資料、安全、公平時調用。

---

## #倫理考量 (#ethicalconsiderations)
**定位**：考慮技術決策的倫理影響——隱私、安全、公平、對使用者的傷害。

**什麼時候用**：處理使用者資料、設計演算法、收集/儲存資訊、權限設計時。

**操作**：
1. 問「這個設計會不會傷害使用者或某個群體」。
2. 檢查資料：只收必要的、安全儲存、清楚告知、可刪除。
3. 檢查演算法：會不會放大偏見/歧視。
4. 預設選最保護使用者的做法(隱私優先、最小權限)。

**檢查清單**：
- □ 我收集的資料是必要的嗎？有沒有過度蒐集？
- □ 敏感資料(密碼/個資/金流)有安全處理嗎？
- □ 這個邏輯會不會對某類使用者不公平？
- □ 預設值是「保護使用者」還是「方便我」？

**程式碼味道**：log 裡印出完整使用者個資/token → 倫理與安全雙重失守。

🔗 **搭配習慣**：#倫理判斷（灰色地帶）、#賽局理論（惡意方）、#當責（為影響負責）

## #倫理勇氣 (#ethicalcourage)
**定位**：敢說出「這樣做不對」，即使不方便、會拖慢進度。

**什麼時候用**：發現安全漏洞/隱私問題/被要求走捷徑、「先上了再說」的壓力下。

**操作**：
1. 發現問題時，明確指出，不因「麻煩/趕時間」而沉默。
2. 用事實與後果說明，而非道德指責。
3. 提出可行的替代方案，不只反對。
4. 對「先這樣之後再修」的危險捷徑說不。

**檢查清單**：
- □ 我有沒有為了省事而對已知問題裝沒看到？
- □ 我指出問題時，有沒有同時給替代方案？
- □ 這個「之後再修」的捷徑，風險我有講清楚嗎？

**程式碼味道**：明知有 SQL injection 風險卻想「先這樣上」→ 該拿出倫理勇氣擋下。

🔗 **搭配習慣**：#倫理考量（看見問題）、#說服技巧（有效表達）、#當責（負責）

## #倫理判斷 (#ethicaljudgment)
**定位**：在沒有標準答案的灰色地帶，做出合乎倫理的權衡判斷。

**什麼時候用**：隱私 vs 功能、效率 vs 公平、商業 vs 使用者利益衝突時。

**操作**：
1. 列出各方利益(使用者、公司、社會)與衝突點。
2. 問「如果這被公開報導，站得住腳嗎」。
3. 找「兼顧多方」的設計，而非二選一。
4. 真要取捨時，傾向保護弱勢/使用者一方。

**檢查清單**：
- □ 各方利益衝突點我看清楚了嗎？
- □ 這個決定攤在陽光下，我站得住腳嗎？
- □ 有沒有兼顧多方的第三條路？

**程式碼味道**：「反正使用者不會發現」這種念頭出現 → 倫理判斷的紅燈。

🔗 **搭配習慣**：#倫理考量（影響面）、#決策樹（攤開權衡）、#詮釋視角（多方角度）
