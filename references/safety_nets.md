# 三條安全網 — 當「做對」失效時的防線

> sentinel 的多數機制在「把事情做對」（協定預想、診斷找根因、病歷防重蹈）。
> 但 vibe coding 真正的暗礁常在「**做錯之後**」與「**做著做著失控**」。
> 這三條安全網補的不是「怎麼寫對」，而是「**寫的過程中不翻船**」——救生艇、誠實的航海日誌、海圖。

---

## 🪂 一、撤退路線（Rollback & Recovery）

**要解的痛**：AI 一路改，改到一半發現方向全錯，卻沒有乾淨回退點，只能整包砸掉重來。

### 動手前：確保退得回去
1. 確認當前是**乾淨可回退點**（git status 乾淨 / 已 commit）。
   - 不乾淨 → 先 commit 或 stash，**別在髒 working tree 上疊新改動**（出事時分不清哪些是新改的）。
2. 大改動前，明示一個**回退錨點**：
   > 「若這次改動失敗，回到 commit `{hash}` / 這個狀態。」
3. 改動**切小步**，每步可獨立驗證、可獨立回退（呼應十二誡第 5 條「每塊 commit」）。

### 動手中：發現方向錯時
4. **不要在錯誤方向上繼續疊補丁救** —— 這是沉沒成本謬誤：「已經改這麼多了，再改一點應該就好」往往讓洞越挖越深。
5. 明確喊停：
   > 「這個方向已經偏了，建議回退到 `{錨點}` 重來，原因是 ___。硬救的話要處理 ___，比回退更費工。」

### 判斷：該回退 vs 該續修
| 訊號 | 決定 |
|---|---|
| 已試 ≥2 個方向仍偏離，且改動越來越髒/越來越多 hack | **回退** |
| 根因明確、只差收尾、改動仍乾淨 | 續修 |
| 已經在「為了讓前面的錯能動」而改更多地方 | **回退**（典型的越救越糟） |

### 接點
- 十二誡第 5 條（每塊 commit）、第 12 條（中斷前 commit + progress.md）。
- 回退過的案例若達夠痛門檻（誤判方向/對抗性除錯）→ 寫病歷。
> 對應習慣：#效用理論（回退vs續修的成本）#批判（誠實面對方向錯了）#當責（不護航自己的錯）

## 🔬 二、證據強度分級（Verification Honesty）

**要解的痛**：AI 很會說「應該可以了」「我認為這樣就對」，但根本沒跑過。把「我覺得」和「我驗證過」混為一談，是 vibe coding 最大的信任殺手。

### 宣稱「完成 / 修好了 / 應該可以了」前，必須標明證據等級

**🟢 已驗證 (verified)**
- 定義：我**實際執行/測試過**，親眼看到預期結果。
- 必附：跑了什麼（指令/測試/操作）、看到什麼輸出。
- 範例：「已驗證：跑 `npm test` 全綠，手動點了表單送出，DB 出現新紀錄。」

**🟡 已檢視 (reviewed)**
- 定義：我**讀過 code 邏輯**確認正確，但**沒實際執行**。
- 必附：明說「尚未實際執行，建議你跑一次確認」。
- 範例：「已檢視：邏輯上應正確，但我沒跑過——建議你執行後確認 edge case。」

**🔴 推測 (assumed)**
- 定義：我**認為應該對**，但沒驗證也沒細看。
- 必附：標 `⚠️推測` + 說明該怎麼驗證。
- 範例：「⚠️推測：這個改法應該能解，但我沒驗證；請跑 ___ 確認。」

### 鐵律
- ❌ 不准用 🟢「已驗證」的語氣，講 🔴 推測的事。
- ❌ 沒跑過就說「我跑過了/測過了」—— 這是最嚴重的失信，比 bug 本身更糟。
- ✅ 不確定能不能驗證時，老實說：「我無法在這裡驗證，需要你 ___。」
- ✅ 環境限制（不能跑、不能連線）導致只能停在 🟡/🔴 → 明講限制，不假裝 🟢。

### 接點
- 強化既有 #可驗證性 #證據基礎。
- 病歷「解法」欄應標證據等級（🟢/🟡/🔴）。
- CLAUDE.md 既有的 `⚠️推測` / `‼️缺資料` 記號。
> 對應習慣：#可驗證性 #證據基礎 #當責（誠實標示）#專業素養（不謊報）

## ⚓ 三、回錨（Anti-Drift）

**要解的痛**：對話一長，AI 忘記最初需求、忘記前面決定、自相矛盾、甚至忘記自己幾輪前才改過的東西。這是 vibe coding **最致命且最獨有**的問題——病歷解決了「跨 session」記憶，但這條解決「**同一 session 內越聊越漂**」。

### 何時回錨（偵測到任一）
- 對話已很長 / context 將滿。
- 要開始一個新的子任務。
- 感覺「我好像忘了原本要幹嘛」。
- 使用者說「等等，我們不是在做 X 嗎」 ← **漂移已發生的警報**。
- 發現自己的說法跟前面矛盾。

### 回錨動作（30 秒，輸出一段簡短對齊）
```
📍 回錨：
· 最初目標：{這個任務/session 最初要達成什麼}
· 目前進度：{已完成什麼、現在在哪一步}
· 待辦：{還剩什麼}
· 有無偏離：{現在做的還對齊最初目標嗎？偏了就明說並拉回}
```

### 防漂移鐵律
- 重大決定、進度寫進 `.claude/progress.md`（呼應十二誡第 12 條），**不靠對話記憶**。
- 自相矛盾時，以「**寫下來的**」為準，不以「剛剛說的」為準。
- context 將滿前**主動回錨** + 提醒使用者：「建議開新 session，並帶上 `progress.md` 讓我接續。」
- 長任務每完成一個子階段就回錨一次，不要等到迷航了才找路。

### 接點
- 十二誡第 12 條（progress.md）、第 1 條（PRD 是合約，回錨時對齊 PRD）。
- 病歷哲學一致：記憶落實體檔案，不靠 LLM 記性。
- 緊急閥：context 將滿也是一種要「止血」的狀況（先存檔再說）。
> 對應習慣：#問對問題（回錨到真目標）#自我覺察（察覺自己漂了）#制定策略（progress 追蹤）#組織結構（progress.md 結構）

## 三者的共同精神

| 安全網 | 像什麼 | 解什麼 |
|---|---|---|
| 撤退路線 | 救生艇 | 改壞了能退回去，不沉船 |
| 證據強度分級 | 誠實的航海日誌 | 不謊報「我到哪了」 |
| 回錨 | 海圖 | 不在長航程中迷失方向 |

> 一句話：sentinel 的其他機制教你**把船造好**，這三條安全網教你**在風浪中不翻船**。
