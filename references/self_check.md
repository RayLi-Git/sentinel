# 淺層紅旗 + 診斷模式（動手中的感測器）

> 本檔是「動手過程中」隨時亮燈的感測器，與「動手前協定」(`preflight.md`) 並行：
> - **動手前**：跑 `preflight.md` 的動手前協定（向內問 + 向外推）。
> - **動手中**：本檔的紅旗隨時偵測你有沒有在淺層打轉/資安僥倖。
> 核心信念：**90% 的卡關不是解法不夠好，是解錯了問題、改錯了地方。**
>
> 註：原本的「動手前自檢 6 問」已融入 `preflight.md` 的「向內」段，不在此重複。

## 淺層打轉紅旗對照（動手中隨時偵測）

| 對同一段 code 改第 3 次 | 問題定義錯了，不是寫法錯 | #問對問題 #差距分析 |

## 紅旗亮起後做什麼（診斷模式步驟）

紅旗亮起 → **退出「改 code 模式」，進入「診斷模式」**：

1. **重述問題**：把問題寫成「預期 X／實際 Y／差距 Z」（#差距分析）。
2. **列假說**：列出 2-3 個可能根因，不要只認定一個（#建立假說）。
3. **排序**：按「合理性 × 驗證成本」排查證順序（#合理性評估）。
4. **隔離驗證**：用最小成本、單一變因驗證哪個假說對（#變數控制 #介入性研究）。
5. **確認根因**：拿到證據確認根因後，才回到「改 code 模式」（#證據基礎）。
6. **🔴升級**：若卡關達重級 → 產出根因樹 HTML（見 html_style_guide.md）。
