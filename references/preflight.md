# 動手前協定 Pre-flight — 動既有 code 之前必跑

> **觸發＝動作，不是階段**：只要你的手「正要動既有的、運作中的 code」（新增會碰到既有的、修 bug、重構、改設定/依賴），**不管在五階段流程的哪一站**，動手前都先跑這套協定。
> 不適用：純獨立新增（全新檔案、不碰既有邏輯）、改註解/文案/typo。
>
> 核心信念：**code 寫得快不是本事，動了不弄壞既有的才是。** 像外科醫師劃刀前那三分鐘——所有會出血的地方，先在腦中開過一遍。

---

## 協定的兩半：向內問 + 向外推

動手前協定融合了兩個方向的思考：

- **向內（我想清楚了嗎）**：問對問題、是症狀還根因、能不能驗證。
- **向外（我會撞到什麼）**：影響範圍、連鎖反應、預想出事。

兩半都跑，才算「動手前想透了」。

---

## 觸發與深度

| 情境 | 跑哪版 |
|---|---|
| 🟡 動既有 code 但範圍小、不碰共用物/資安面 | **精簡版**（一輪掃過，30 秒內） |
| 🔴 牽連廣 / **碰到共用物** / **碰到資安面** | **完整版**（逐點展開，產出協定清單） |

**自動升級成完整版的條件（碰到任一）：**
- 共用物：共用函式 / 共用狀態 / 資料庫 / API 契約 / 共用型別定義 / 設定檔
- 資安面：認證 / 權限 / 個資(PII) / 金流 / 機密

---

## 精簡版（🟡，一輪掃過）

動手前快速自問，全部「無問題/無風險」才直接動手：

**向內：**
1. 問對問題：我要動的，是真問題還是腦補的？
2. 症狀vs根因：（若修 bug）我動的是根源處還是發生處？

**向外：**
3. 影響範圍：這會碰到既有的什麼？誰在用我要動的東西？
4. 連鎖風險：碰到的會不會壞？
5. 安全＋驗證：怎麼做不弄壞既有？改完怎麼證明真的好了？

> 任一點浮現「會碰共用物 / 資安面」或「講不清楚」→ 立刻升完整版。

## 完整版（🔴，逐點展開）

### 向內 ── 我想清楚了嗎

**A. 問對問題**
- 一句話複述「真正要達成的結果」。對不上 → 停。
- （修 bug）症狀 vs 根因：我動的是根源處還是錯誤發生處？
- （修 bug）最小重現：能說出「預期X／實際Y／差Z」嗎？講不出不准動手。
> 習慣：#問對問題 #差距分析 #複雜因果關係

### 向外 ── 我會撞到什麼

**B. 影響範圍預測（blast radius）**
- 會改到哪些檔 / 函式 / 元件？
- 會讀寫哪些共用狀態 / 全域變數？
- 會動到 DB schema / API 契約 / 型別定義嗎？
- **誰在呼叫我要動的東西**？（往上游找呼叫者——最常漏）
> 習慣：#網絡分析 #系統描繪

**C. 連鎖反應預測**
- B 列出的每個被碰到的東西，**會不會壞**？
- 有沒有別處依賴它「原本的行為」？
- 改共用函式 → 其他呼叫處會不會被波及？
- 改資料結構 → 既有資料 / 序列化 / 快取會不會不相容？
> 習慣：#複雜因果關係 #湧現

**D. 安全路徑設計**
- 能不能**新增而非修改**（加新函式 vs 改舊函式）？
- 需不需要相容層 / 漸進遷移 / feature flag？
- 改動順序怎麼排，讓每步可獨立驗證、可回退？
- 哪些既有測試必須仍然通過？
> 習慣：#制定策略 #限制條件 #最小權限

**E. Pre-mortem 風險預想（假設它已經出事了）**
- 想像上線後出事，回頭看最可能哪裡爆？（邊界/並發/空值/大量資料/網路失敗）
- 哪個 edge case 我沒打算處理，但其實會發生？
- 上游/第三方回傳非預期，我這裡會怎樣？
- 出錯時 fail closed（安全）還是 fail open（危險）？
- 改完怎麼**證明**真的好了，而非看起來好了？（可驗證）
> 習慣：#批判 #賽局理論 #抽樣 #可驗證性
> 🔴 碰資安面 → 同時跑資安五習慣（#不信任輸入 #認證授權 #最小權限 #敏感資料 #預設安全）

**F. Claude 自主補充**
> 「除了上面，**這個具體情境**我覺得還該事先想的是 ___」

主動指出此任務特有、通用清單沒涵蓋的考量（時區→夏令時間；金額→浮點精度；檔案→並發寫入…）。

## 協定清單產出（🔴 完整版才產）

完整版後，動手前先輸出簡短清單給使用者看：

```
🛫 動手前協定：{要動的東西}
向內｜問對問題：{真正目標}（修bug另列 症狀vs根因）
② 影響範圍：{會碰到的檔/函式/狀態}
③ 連鎖風險：{可能被波及的，及為什麼}
④ 安全做法：{怎麼做不弄壞既有，改動順序}
⑤ 預想風險：{最可能出錯點 + 怎麼防 + 怎麼驗證}
⑥ 額外考量：{Claude 針對此情境補充}
→ 確認方向後開始動手
```

> 若清單揭露重大風險/取捨，依使用者偏好給 ≥4 個做法選項讓使用者選方向。

## 與其他機制的銜接

- 協定完成 → 動手 → 出問題進「診斷」→ 夠痛則寫病歷。
- 協定 E 若預測到某風險、後來真發生 → 病歷標「協定已預測到」，強化 patterns。
- 協定 E 若**沒**預測到、後來爆了 → 復盤檢討「為何協定沒抓到」，高價值 #自我覺察。
- PRD 任務：協定 B 影響範圍應對齊 PRD 章節，不擴張 PRD 沒寫的（呼應十二誡 §10 YAGNI）。
- 淺層紅旗 / 資安紅旗（見 `self_check.md`）是獨立感測器，動手中隨時亮燈，與本協定並行。
