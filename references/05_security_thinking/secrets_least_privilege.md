# 資安思考 / 機密與最小權限（code 語境）

> 主分類色：深青藍 `#1E3A8A`。保護秘密、限縮權力——出事時把傷害降到最小。

---

## #機密管理 (#secretsmanagement)　⭐高頻紅旗
**定位**：密鑰、token、密碼、連線字串——絕不進 code、log、git、前端、錯誤訊息。

**什麼時候用**：用到任何 API key、資料庫密碼、token、憑證、加密金鑰時。**vibe coding 最常翻車處**。

**操作**：
1. 機密一律從環境變數/密鑰管理服務讀，不寫死在 code。
2. 確認機密不會被 commit 進 git(用 .gitignore、掃描 .env)。
3. log 與錯誤訊息絕不印出機密(連部分都不行)。
4. 機密不送到前端、不放 URL 參數、不存 localStorage。
5. 懷疑外洩 → 立刻輪替(rotate)，不要只是刪掉。

**檢查清單**：
- □ 有沒有任何金鑰/密碼被寫死在 code 裡？
- □ `.env` / 含機密的檔有沒有進 .gitignore？
- □ 我的 log 會不會印出 token/密碼/個資？
- □ 機密有沒有跑到前端/URL/localStorage？
- □ (歷史)git 紀錄裡有沒有曾經 commit 過機密？

**程式碼味道**：`const apiKey = "sk-..."` 寫死、或 `console.log(user)` 印出含 token 的物件 → 機密外洩，且 git 一旦記錄就永久存在。

🔗 **搭配習慣**：#敏感資料（資料保護）、#不信任輸入（邊界）、#當責（外洩誠實處理）

## #最小權限 (#leastprivilege)
**定位**：每個元件、使用者、token 只給「剛好夠用」的權限，不多給。

**什麼時候用**：設計權限、發 token、設資料庫帳號、設雲端 IAM、第三方授權時。

**操作**：
1. 預設「什麼都不給」，再逐項加上必要的。
2. token/帳號的權限範圍縮到最小(只讀就別給寫、單表就別給全庫)。
3. 設定過期時間，不發永久憑證。
4. 定期檢視並收回不再需要的權限。

**檢查清單**：
- □ 這個 token/帳號的權限，有沒有超出它實際需要的？
- □ 我是不是為了省事給了 admin/全權限？
- □ 憑證有過期機制嗎，還是永久有效？
- □ 出事時，這個權限能造成多大破壞？(爆炸半徑)

**程式碼味道**：圖方便給服務帳號 admin 權限、token 設永不過期 → 一旦外洩，爆炸半徑最大化。

🔗 **搭配習慣**：#預設安全（預設關閉）、#認證授權（授權設計）、#攻擊面思考（縮小破壞）

## #敏感資料 (#sensitivedata)
**定位**：個資、金流、健康等敏感資料的蒐集、儲存、傳輸全程最小化與加密。

**什麼時候用**：處理使用者個資、金流、密碼、任何隱私資料時。

**操作**：
1. 蒐集最小化：不需要的就別收(接 #倫理考量)。
2. 傳輸加密(HTTPS/TLS)，儲存加密(at rest)。
3. 密碼用單向雜湊(bcrypt/argon2)，絕不明文或可逆加密。
4. 提供刪除機制，遵守資料保護法規。

**檢查清單**：
- □ 我收的敏感資料都是必要的嗎？
- □ 傳輸和儲存有沒有加密？
- □ 密碼是雜湊還是明文/可逆？
- □ 使用者能要求刪除他的資料嗎？

**程式碼味道**：密碼用明文或 MD5 存、敏感資料走 HTTP → 資料保護失守。

🔗 **搭配習慣**：#機密管理（保護秘密）、#倫理考量（隱私）、#最小權限（存取控制）
